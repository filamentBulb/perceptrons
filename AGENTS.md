# Repository Guidelines

## Project Overview

This repo is a TanStack Start prototype for **Runway AI CFO**: a startup finance dashboard that combines cloud pricing, revenue, banking, runway, and AI what-if analysis. The cloud pricing side is backed by checked-in public pricing datasets generated by `scraper.py`; the AI CFO chat is a server route that calls IBM watsonx.ai.

The `README.md` still contains TanStack starter leftovers and Anthropic references. Treat `AGENTS.md`, `CLAUDE.md`, and the source code as the current source of truth until the README is refreshed.

## Project Structure

- `src/routes/` - TanStack Router file routes. Main screens are `/` (`index.tsx`), `/connect`, `/dashboard`, `/ai-cfo`, and `/about`. Server endpoints are `/api/ai-cfo/chat` and `/mcp`.
- `src/routeTree.gen.ts` - generated route tree. Do not edit it manually.
- `src/components/` - shared UI, header/footer, and shadcn-style primitives in `src/components/ui/`.
- `src/lib/runway-store.ts` - TanStack Store for connected source IDs, persisted to `localStorage` as `runway-ai-cfo-state`.
- `src/lib/provider-types.ts` - shared types for provider mocks, pricing line items, forecasts, and scenarios.
- `src/data/cloud-pricing/` - checked-in pricing dataset. `inputs.json` is the editable scenario config; `catalogue.json` and `estimate.json` are generated outputs.
- `src/data/providers.ts` - adapts generated cloud pricing totals into UI provider records.
- `src/styles.css` - Tailwind v4 entry, theme variables, reusable shells, and global styling. Biome intentionally excludes this file.
- `scraper.py` - Python scraper/generator for account-state snapshots and public cloud pricing estimates.
- `test_scraper_forecast.py` - Python `unittest` coverage for scraper forecasting and public pricing behavior.

## Commands

- `pnpm install` - install dependencies from `pnpm-lock.yaml`.
- `pnpm dev` - run Vite dev server on `PORT`, default `3000`.
- `pnpm build` - production build through Vite/TanStack Start/Nitro; output is `.output/`. Husky runs this on pre-commit.
- `pnpm preview` - preview the built app.
- `pnpm test` - run Vitest once.
- `pnpm lint` - run Biome lint.
- `pnpm format` - run Biome format.
- `pnpm check` - run Biome format and lint validation.
- `python -m unittest test_scraper_forecast.py` - run scraper tests.
- `python scraper.py --pricing-estimates --gcp-api-key-file=.gcp_api_key` - regenerate cloud pricing outputs from public pricing sources.

## Application Model

Source IDs are `aws`, `gcp`, `azure`, `cloudflare`, `stripe`, and `banking`. Forecast and AI CFO features require at least one cloud source plus at least one banking/revenue source; this logic lives in `hasRequiredForecastSources()`.

The home route (`src/routes/index.tsx`) is the richest app surface: integration cards, connection flow, pricing dataset summary, top line items, scenario cards, and forecast views. `src/routes/dashboard.tsx` is a simpler static dashboard mock. `src/routes/ai-cfo.tsx` posts chat messages to `src/routes/api.ai-cfo.chat.ts`.

The AI route reads `IBM_API_KEY` plus either `IBM_WATSONX_PROJECT_ID`/`IBM_PROJECT_ID`/`WATSONX_PROJECT_ID` or `IBM_WATSONX_SPACE_ID`/`WATSONX_SPACE_ID`. It caches IBM IAM tokens and tries a comma-separated `IBM_WATSONX_MODEL_ID` list before built-in fallback models.

`src/routes/mcp.ts` exposes an MCP server with an `addTodo` tool. `src/utils/mcp-handler.ts` bridges each POST through an in-memory MCP transport pair. Todo state is in `src/mcp-todos.ts`.

## Coding Style

Use TypeScript and React function components. Follow local patterns before adding abstractions. Prefer the `#/*` source alias for imports from `src/`; several older files still use relative imports, so avoid broad import-only churn.

Biome is the source of truth for TypeScript formatting: tabs and double quotes. Route and MCP files with single quotes are existing style debt, not a new convention. Use `PascalCase` for components, `camelCase` for functions/variables, and `useName` for hooks.

For UI, prefer existing CSS variables and classes from `src/styles.css` (`--sea-ink`, `--lagoon-deep`, `--surface-strong`, `--line`, `page-wrap`, `island-shell`, `display-title`) over raw color-heavy Tailwind. Dark mode is controlled by `.dark` on `<html>` and `ThemeToggle`.

Files prefixed with `demo` or `demo.` are scaffold leftovers unless currently imported. Remove or replace them when they become misleading.

## Data Guidelines

Do not hand-edit `catalogue.json` or `estimate.json` unless the task is explicitly about fixture repair. Update `inputs.json` for scenario assumptions, then regenerate with `scraper.py` when changing pricing behavior. The generated data is consumed through `src/data/cloud-pricing/index.ts`; keep type changes aligned with `src/lib/provider-types.ts`.

If source IDs, provider labels, or baseline finance numbers change, check all duplicated contexts: `index.tsx`, `connect.tsx`, `dashboard.tsx`, and `api.ai-cfo.chat.ts`.

## Testing Guidelines

Add focused Vitest/Testing Library tests for user-facing React behavior, store helpers, utilities, and server route logic. Co-locate tests as `*.test.ts` or `*.test.tsx`, or use a nearby `__tests__/` directory. Run `pnpm test` before handing off substantive TypeScript changes.

For scraper changes, run `python -m unittest test_scraper_forecast.py`. The scraper reaches public provider APIs when regenerating data, so do not run regeneration casually when network access, credentials, or checked-in JSON churn are not part of the task.

## Security And Configuration

Keep API keys and local secrets in ignored files or the shell environment. `.env` is ignored; `.gcp_api_key` is also intentionally local. Never log raw provider credentials, IBM IAM tokens, request bodies containing user finance context, or provider API responses with sensitive account data.

Required AI CFO env:

- `IBM_API_KEY`
- one of `IBM_WATSONX_PROJECT_ID`, `IBM_PROJECT_ID`, `WATSONX_PROJECT_ID`, `IBM_WATSONX_SPACE_ID`, or `WATSONX_SPACE_ID`

Optional AI CFO env:

- `IBM_WATSONX_URL` (defaults to `https://eu-de.ml.cloud.ibm.com`)
- `IBM_WATSONX_MODEL_ID` (comma-separated fallback list)
- `IBM_WATSONX_API_VERSION` (defaults to `2024-05-31`)

## Commit And PR Notes

Use short, imperative commit messages scoped to one logical change. For PRs, include a short summary, verification commands, linked issues when applicable, screenshots or recordings for UI changes, and any new environment variables without secrets. Husky runs `pnpm build`; fix build failures instead of bypassing hooks.
